Q2. LFU Cache

### Problem
Design and implement a data structure for a Least Frequently Used (LFU) cache.

#### Constraints
- 1 <= capacity <= 104 
- 0 <= key <= 105 
- 0 <= value <= 109 
- At most 2 * 105 calls will be made to get and put.

### Implementation
To achieve O(1) for both `get()` and `put()`:
- Maintain
: `key → {value, frequency}`
: `frequency → list of keys` (LRU order inside same frequency)
: `key → iterator` (to remove from list in O(1))
: `minFreq tracker` (to know which frequency to evict from)

Code
```c
class LFUCache {
private:
    int cap;
    int minFreq;
    
    unordered_map<int, pair<int,int>> keyData; 
    // key -> {value, frequency}
    
    unordered_map<int, list<int>> freqList; 
    // frequency -> list of keys (LRU order)
    
    unordered_map<int, list<int>::iterator> keyIter; 
    // key -> iterator in freqList
    
    void updateFrequency(int key) {
        int freq = keyData[key].second;
        
        freqList[freq].erase(keyIter[key]);
        
        if (freqList[freq].empty()) {
            freqList.erase(freq);
            if (minFreq == freq)
                minFreq++;
        }
        
        keyData[key].second++;
        freq++;
        
        freqList[freq].push_front(key);
        keyIter[key] = freqList[freq].begin();
    }

public:
    LFUCache(int capacity) {
        cap = capacity;
        minFreq = 0;
    }
    
    int get(int key) {
        if (cap == 0 || keyData.find(key) == keyData.end())
            return -1;
        
        updateFrequency(key);
        return keyData[key].first;
    }
    
    void put(int key, int value) {
        if (cap == 0)
            return;
        
        if (keyData.find(key) != keyData.end()) {
            keyData[key].first = value;
            updateFrequency(key);
            return;
        }
        
        if (keyData.size() == cap) {
            int evictKey = freqList[minFreq].back();
            freqList[minFreq].pop_back();
            
            if (freqList[minFreq].empty())
                freqList.erase(minFreq);
            
            keyData.erase(evictKey);
            keyIter.erase(evictKey);
        }
        
        keyData[key] = {value, 1};
        freqList[1].push_front(key);
        keyIter[key] = freqList[1].begin();
        minFreq = 1;
    }
};
```
### How it works
```bash
Frequency buckets
freq = 1 → [most recent ... least recent]
freq = 2 → [...]
freq = 3 → [...]
```

Inside each frequency list:
- Front = most recently used
- Back = least recently used

So when we evict:
`evict from freqList[minFreq].back()`

That guarantees least frequency, if tie → least recently used

### Complexity
| Operation   | Time        |
| ----------- | ----------- |
| get         | 0(1)        |
| put         | 0(1)        |

All operations are hash lookup + list insert/remove.
